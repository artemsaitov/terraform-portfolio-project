# terraform-portfolio-project
Next.js Portfolio Website Deployment on AWS with Terraform
Project Overview

This project demonstrates how to deploy a Next.js portfolio website on AWS using Infrastructure as Code (IaC) with Terraform.

The goal is to host a modern static website using a scalable, highly available, and cost-effective cloud architecture that delivers fast performance for users worldwide.

The infrastructure provisions Amazon S3 for static hosting and Amazon CloudFront as a global CDN, ensuring low latency and reliable access to the website.

This project was built as part of a Cloud Engineering learning journey and portfolio development.
Scenario

Client: James Smith — Freelance Web Designer

James has designed a modern single-page portfolio website using Next.js and needs a reliable platform to host it.

The website should be:

Highly Available – Accessible worldwide with minimal downtime

Scalable – Able to handle increasing traffic

Cost-Effective – Avoid unnecessary infrastructure costs

Fast Loading – Low latency for global users

The solution uses AWS cloud services and Terraform to deploy the infrastructure automatically.

Architecture Overview

The website is deployed using a simple and scalable cloud architecture:

Users
  ↓
Amazon CloudFront (CDN)
  ↓
Amazon S3 (Static Website Files)
Key Components

Amazon S3

Stores the static files generated by the Next.js build process.

Amazon CloudFront

Acts as a global Content Delivery Network (CDN), caching and delivering content from edge locations around the world to reduce latency.

Terraform

Manages infrastructure provisioning using Infrastructure as Code, allowing reproducible and automated deployments.

Next.js is an open-source React framework maintained by Vercel that enables developers to build fast and scalable web applications.

In this project, Next.js is used to generate a static version of the website, which can then be hosted on cloud storage and delivered globally through a CDN.

Deployment Steps

1. Build the Next.js Application

Generate the static files for deployment.
npm run build
npm run export

This creates production-ready files in the out/ directory.

2. Provision Infrastructure with Terraform

*  Navigate to the Terraform directory:
cd terraform-nextjs

* Initialize Terraform:
terraform init

* Review the infrastructure plan:
terraform plan

* Deploy the infrastructure:
terraform apply

Terraform will create the required AWS resources such as the S3 bucket and CloudFront distribution.

3. Upload the Website to Amazon S3

Sync the generated static files to the S3 bucket:
aws s3 sync ./out s3://your-unique-bucket-name
